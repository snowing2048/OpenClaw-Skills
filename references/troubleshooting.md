# Browser Agent 常见问题

## 点击失败

### 问题：click_text.py 找不到文字

**原因**：文字不在当前可见区域

**解决**：
1. 先用滚轮滚动到目标区域
2. 再次尝试点击
3. 如果还是不行，截图给模型重新分析

```bash
# 先滚动
py scroll.py down 3

# 再尝试点击
py click_text.py "目标按钮"
```

---

## 截图问题

### 问题：截图是空的/黑的

**原因**：页面还没加载完成

**解决**：
```bash
# 多等一会儿
browser action=act profile=openclaw targetId=<tab_id> kind=wait timeMs=5000
```

---

## 输入问题

### 问题：输入变成中文了

**原因**：输入法没切换到英文

**解决**：
1. 手动切换到英文输入法
2. 或者在输入前先按一下 `Shift` 键切换

---

## 登录问题

### 问题：遇到登录弹窗

**处理流程**：

```
1. 截图
2. 发给模型识别
3. 模型确认是登录弹窗
4. → 停止操作，提醒用户

提醒内容：
"该网站需要登录才能继续操作。你是否有账户？需要我帮你登录吗？"
```

**不要**：
- ❌ 反复尝试输入
- ❌ 一直点击登录按钮
- ❌ 不告诉用户

---

## 验证码问题

### 问题：遇到验证码（滑块、图形等）

**处理流程**：

```
1. 截图
2. 发给模型识别
3. 模型确认是验证码
4. → 停止操作，提醒用户

提醒内容：
"该网站需要验证，你可能需要手动完成验证后告诉我继续。"
```

---

## 页面问题

### 问题：页面加载失败/404

**解决**：
1. 截图确认
2. 检查 URL 是否正确
3. 尝试重新打开
4. 如果反复失败，提醒用户

---

## 浏览器问题

### 问题：浏览器没响应

**解决**：
1. 尝试刷新页面
2. 关闭并重新打开浏览器
3. 检查网络连接

---

## 决策流程图

```
开始操作
   ↓
截图 ──→ 发给模型识别
   ↓
模型返回页面内容
   ↓
是登录弹窗？ ──是──→ 提醒用户，停止
   ↓否
是验证码？ ───是──→ 提醒用户，停止
   ↓否
是错误页面？ ──是──→ 尝试刷新或换方式
   ↓否
执行操作（点击/输入/滚动）
   ↓
截图验证
   ↓
操作成功？ ──否──→ 重试或换方式
   ↓是
继续下一步 或 完成
```

## 快速检查清单

操作前检查：
- [ ] 页面加载完成了吗？
- [ ] 目标按钮在可见区域吗？
- [ ] 输入法是英文吗？

操作后检查：
- [ ] 截图确认结果了吗？
- [ ] 模型识别成功了吗？
- [ ] 需要提醒用户吗？
